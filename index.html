What this is

Learn a backend by walking through a real one

This is a working reference project — a small user-registration API built with Express and MongoDB, laid out the way a production service actually gets organised: routes, middleware, validation, controllers, services and models, each with one job. Click any file in the tree to read what it does and why it lives where it does. Then use the request simulator below to watch a real request travel through every one of those layers, in order, the same path it would take on an actual server.
