Why this site exists

Stop putting everything in server.js

Most beginners write their first API in one file. It works, right up until that file is 600 lines of routes, database calls, password hashing, and error handling tangled together. This site teaches separation of concerns: the practice of giving every part of your backend exactly one job, in its own file, with a name that tells you what it does before you even open it.

Below, you'll see a real signup endpoint built two ways — crammed into one file, and split across the folders professional Node projects actually use — followed by the naming rules that keep a growing codebase readable, and an explorer where you can inspect every file yourself.

01 · The problem, in code

One file doing five jobs vs. five files doing one job each

Same feature — user registration — written both ways. The left version works. The right version is the one you can still read in six months.

server.js — everything, one file
app.post('/register', async (req, res) => {
  const { username, email, password } = req.body;
  if (!username || !email || !password) {
    return res.status(400).json({ error: 'Missing fields' });
  }
  const existing = await db.collection('users')
    .findOne({ email });
  if (existing) {
    return res.status(400).json({ error: 'Email taken' });
  }
  const hashed = await bcrypt.hash(password, 10);
  const user = await db.collection('users').insertOne({
    username, email, password: hashed
  });
  const token = jwt.sign({ id: user.insertedId }, SECRET);
  res.status(201).json({ token });
});

Validation, database access, hashing, and token creation are all inline. Testing any one piece means running the whole route.

split by responsibility
// routes/auth.routes.js
router.post('/register', validateSchema(userSchema), registerUser);

// controllers/authController.js
export const registerUser = async (req, res) => {
  const user = await registerNewUser(req.body);
  res.status(201).json({ success: true, data: user });
};

// services/userService.js
export const registerNewUser = async (data) => {
  const hashed = await bcrypt.hash(data.password, 10);
  return User.create({ ...data, password: hashed });
};

Each file can be read, tested, and reused on its own. Change how passwords are hashed and only userService.js moves.
